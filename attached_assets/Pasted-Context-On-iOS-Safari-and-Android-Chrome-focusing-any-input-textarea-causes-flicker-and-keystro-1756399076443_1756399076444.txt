Context
On iOS Safari and Android Chrome, focusing any <input>/<textarea> causes flicker and keystrokes don’t appear. In client/src/pages/MobileApp.tsx, the login and other form inputs use inline styles with touchAction: 'manipulation' and WebkitAppearance: 'none', which interfere with the keyboard and text entry on mobile (these appear in the Login inputs and other form fields). 

Goals (apply all):

Remove problematic inline props everywhere

Find all <input> and <textarea> elements in client/src/pages/MobileApp.tsx and any other form files.

Delete touchAction: 'manipulation' and WebkitAppearance: 'none' from inline style={...} objects.

Keep the rest of the styling intact.

Make inputs mobile-friendly

For text inputs/areas, add non-invasive mobile attrs:

inputMode="text"
autoCapitalize="sentences"
autoCorrect="on"
spellCheck={true}


Ensure all inputs/textareas are controlled (they already are); don’t mix defaultValue with value.

Height/viewport stabilization

In MobileApp.tsx the container style uses height: '100vh' in styles.container. Replace it with:

minHeight: '100dvh'


(leave other style rules unchanged). This avoids iOS keyboard resize jumps.

Tiny CSS safety shim (optional if present already)

In client/src/index.css, ensure:

html, body, #root { height: 100%; }
/* Optional footer padding if you have a fixed bottom bar */
.safe-bottom { padding-bottom: env(safe-area-inset-bottom, 0); }


If a bottom bar overlaps inputs, add className="safe-bottom" to the scrollable content wrapper or adjust its padding-bottom by calc(env(safe-area-inset-bottom, 0) + 60px).
